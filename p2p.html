P2P File Sharing Software

Aug 2018 - Oct 2018

BitTorrent is a popular P2P protocol for file distribution. Among its interesting features, I implemented the choking-unchoking mechanism which is one of the most important features of BitTorrent.

The main functionality of this software is to sharing files between peers. Unlike client-server architecture, there is no central server for processing requests in a P2P architecture. One peer could both become a server and a client. I used Java based on BitTorrent protocol to complete this software. I divided my program into 3 parts, first part is called client, which is used to send messages to others. The second part is called server part. This part is aimed to receive messages which sent from othersâ€™ client. The third part, is called a manager part. You know, when a new peer joins the network, all of the old peers in this network will generate at least 2 threads for the new one. One for client and one for server. Normally there are lots of peers in one p2p network. Therefore, for one peer, it needs to establish many threads in order to keep connections to every other peer. So I designed a manager part to organize and manage these threads to reduce the pressure of the server part. It could receive messages from server part, and choose an appropriate client thread to push the message. This is originally the server part work, now the server only need to do is to receiving and handling messages sent from other peers. After that it could just give the processed message to the manager and move on to the next messages.
